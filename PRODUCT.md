# FocusOS Product Context

FocusOS is Matthew's operational source of truth: command center, capture surface, review loop, and Hermes cockpit.

## Audience

- Primary user: Matthew.
- Secondary operator: Hermes, acting as execution monitor and coordination layer.
- Use context: Mac and iPhone, school transition, revenue work, personal operations, high context switching.

## Core Jobs

- Capture obligations, tasks, thoughts, and signals fast.
- Convert noise into clear next actions.
- Surface blockers, overdue work, stalled projects, and attention leaks.
- Keep today's work legible across tasks, habits, calendar, capture, and assistant summary.
- Make review loops easy enough to use daily.

## Product Lane

FocusOS is product UI, not marketing UI. It should feel like a quiet operating cockpit: dense enough to be useful, restrained enough to reduce stress, structured enough that Hermes can reason over it.

## Success Criteria

- Matthew can see what matters today within seconds.
- Mobile layout stays usable without hiding important actions.
- New surfaces reduce cognitive load rather than adding dashboard ornament.
- Assistant/Hermes outputs become actionable, inspectable, and sync-safe.
- Every meaningful feature improves capture speed, priority clarity, follow-through, blocker visibility, review quality, mobile use, sync reliability, revenue support, or stress reduction.

## Non-Goals

- Ornamental analytics without decisions attached.
- Generic productivity clone patterns.
- Fancy dashboards that do not change execution behavior.
- Broad architecture changes without reliability or workflow payoff.
- Visual novelty that makes daily scanning harder.
